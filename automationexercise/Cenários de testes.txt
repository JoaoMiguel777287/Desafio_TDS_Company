
## Cenários de Teste Manual: Login e Cadastro (automationexercise.com)

### Objetivo Geral

Validar a robustez, usabilidade e conformidade das funcionalidades de registro de novos usuários e autenticação no sistema, garantindo uma experiência segura e consistente.

---

### Funcionalidade: Cadastro de Usuário (Signup)

**URL Base para Cadastro:** `https://www.automationexercise.com/login` (clicar em 'Signup')

---

#### Cenários Positivos

1.  **Nome do Caso de Teste:** Cadastro Completo de Novo Usuário com Sucesso
    * **Objetivo:** Verificar se um novo usuário pode se registrar com sucesso preenchendo todos os campos obrigatórios e opcionais.
    * **Pré-condições:** Acesso à página de Cadastro.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  No formulário "New User Signup!", preencher "Name" e "Email Address" com dados únicos e válidos.
        3.  Clicar em "Signup".
        4.  Na página "ENTER ACCOUNT INFORMATION", preencher todos os campos obrigatórios (Título, Senha, Data de Nascimento).
        5.  Preencher campos opcionais (Primeiro Nome, Sobrenome, Empresa, Endereço 1, Endereço 2, País, Estado, Cidade, CEP, Número de Celular).
        6.  Marcar/Desmarcar checkboxes "Sign up for our newsletter!" e "Receive special offers from our partners!" (testar ambos os estados).
        7.  Clicar em "Create Account".
    * **Resultados Esperados:**
        * Usuário é redirecionado para a página "ACCOUNT CREATED!".
        * Mensagem "Account Created!" é exibida em destaque e verde.
        * Botão "Continue" é exibido.
        * Ao clicar em "Continue", o usuário é redirecionado para a Home Page logado (nome de usuário exibido no cabeçalho).

2.  **Nome do Caso de Teste:** Cadastro Básico de Novo Usuário com Sucesso
    * **Objetivo:** Verificar se um novo usuário pode se registrar com sucesso preenchendo apenas os campos estritamente obrigatórios.
    * **Pré-condições:** Acesso à página de Cadastro.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  No formulário "New User Signup!", preencher "Name" e "Email Address" com dados únicos e válidos.
        3.  Clicar em "Signup".
        4.  Na página "ENTER ACCOUNT INFORMATION", preencher **apenas** os campos obrigatórios.
        5.  Clicar em "Create Account".
    * **Resultados Esperados:** Semelhantes ao cenário 1.

---

#### Cenários Negativos 

3.  **Nome do Caso de Teste:** Tentativa de Cadastro com Email Já Registrado
    * **Objetivo:** Garantir que o sistema não permite múltiplos cadastros com o mesmo endereço de e-mail e informa o usuário.
    * **Pré-condições:** Um usuário já registrado no sistema.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  No formulário "New User Signup!", preencher "Name" e "Email Address" com um e-mail **já cadastrado**.
        3.  Clicar em "Signup".
    * **Resultados Esperados:**
        * Mensagem de erro "Email Address already exist!" é exibida abaixo do campo de e-mail ou em uma área de destaque.
        * O usuário permanece na página de Cadastro.

4.  **Nome do Caso de Teste:** Cadastro com Campos Obrigatórios Vazios (Parcial)
    * **Objetivo:** Verificar se a validação de campos obrigatórios funciona corretamente.
    * **Pré-condições:** Acesso à página de Cadastro.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  No formulário "New User Signup!", preencher "Name" mas deixar "Email Address" vazio. OU preencher "Email Address" e deixar "Name" vazio.
        3.  Clicar em "Signup".
    * **Resultados Esperados:**
        * Mensagens de erro apropriadas são exibidas para os campos vazios (ex: "Please fill out this field").
        * O usuário permanece na página.

5.  **Nome do Caso de Teste:** Cadastro com Campos Obrigatórios Vazios (Página 2)
    * **Objetivo:** Verificar a validação de campos obrigatórios na segunda etapa do cadastro.
    * **Pré-condições:** Acesso à página "ENTER ACCOUNT INFORMATION".
    * **Passos:**
        1.  Completar a primeira etapa do cadastro (Name e Email).
        2.  Na página "ENTER ACCOUNT INFORMATION", deixar um ou mais campos obrigatórios (ex: Senha, Data de Nascimento) vazios.
        3.  Clicar em "Create Account".
    * **Resultados Esperados:**
        * Mensagens de erro apropriadas são exibidas para os campos vazios.
        * O usuário permanece na página "ENTER ACCOUNT INFORMATION".

6.  **Nome do Caso de Teste:** Cadastro com Formato de Email Inválido
    * **Objetivo:** Verificar se o sistema valida o formato do endereço de e-mail.
    * **Pré-condições:** Acesso à página de Cadastro.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  Preencher "Name".
        3.  Preencher "Email Address" com formatos inválidos (ex: "teste@.com", "teste@com", "teste", "teste.com").
        4.  Clicar em "Signup".
    * **Resultados Esperados:**
        * Mensagem de erro de validação de e-mail é exibida (geralmente do navegador/HTML5, mas validar a usabilidade).
        * O usuário permanece na página.

7.  **Nome do Caso de Teste:** Validação de Formatos para Campos Opcionais (e.g., Celular, CEP)
    * **Objetivo:** Verificar se campos opcionais com formatos específicos são validados corretamente.
    * **Pré-condições:** Acesso à página "ENTER ACCOUNT INFORMATION".
    * **Passos:**
        1.  Completar a primeira etapa do cadastro.
        2.  Na página "ENTER ACCOUNT INFORMATION", preencher campos como "Mobile Number" ou "Zipcode" com formatos inválidos (ex: letras em número de celular, CEP com formato incorreto).
        3.  Preencher todos os outros campos obrigatórios e clicar em "Create Account".
    * **Resultados Esperados:**
        * Mensagens de erro de validação apropriadas são exibidas para os campos com formato incorreto.
        * O usuário permanece na página.

8.  **Nome do Caso de Teste:** Back/Forward Browser durante o Cadastro
    * **Objetivo:** Verificar o comportamento do sistema ao usar os botões de voltar/avançar do navegador durante o fluxo de cadastro.
    * **Pré-condições:** Acesso à página de Cadastro.
    * **Passos:**
        1.  Iniciar o cadastro.
        2.  Após a primeira etapa, na página "ENTER ACCOUNT INFORMATION", clicar no botão "Voltar" do navegador.
        3.  Verificar estado da página anterior.
        4.  Clicar "Avançar" do navegador.
        5.  Tentar prosseguir com o cadastro.
    * **Resultados Esperados:**
        * A navegação deve ser consistente. Idealmente, os dados preenchidos devem ser mantidos ou o fluxo deve ser reiniciado de forma controlada.

### Funcionalidade: Login de Usuário (Login)

**URL Base para Login:** `https://www.automationexercise.com/login`

---

#### Cenários Positivos (Happy Path)

1.  **Nome do Caso de Teste:** Login com Credenciais Válidas (Usuário Existente)
    * **Objetivo:** Verificar se um usuário existente pode fazer login com sucesso usando credenciais corretas.
    * **Pré-condições:** Um usuário válido já cadastrado no sistema (ou criado em um pré-requisito).
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  No formulário "Login to your account", preencher "Email Address" e "Password" com as credenciais válidas.
        3.  Clicar em "Login".
    * **Resultados Esperados:**
        * Usuário é redirecionado para a Home Page.
        * O nome do usuário logado é exibido no cabeçalho (e.g., "Logged in as UserName").
        * Funcionalidades pós-login (ex: "Delete Account", "Logout") estão visíveis.

---

#### Cenários Negativos / Edge Cases

2.  **Nome do Caso de Teste:** Login com Email Não Registrado
    * **Objetivo:** Garantir que o sistema nega o acesso para e-mails não cadastrados e informa o usuário.
    * **Pré-condições:** Acesso à página de Login.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  Preencher "Email Address" com um e-mail **não registrado**.
        3.  Preencher "Password" com qualquer senha.
        4.  Clicar em "Login".
    * **Resultados Esperados:**
        * Mensagem de erro "Your email or password is incorrect!" é exibida.
        * O usuário permanece na página de Login.

3.  **Nome do Caso de Teste:** Login com Senha Incorreta
    * **Objetivo:** Garantir que o sistema nega o acesso para senhas incorretas, mesmo para e-mails válidos.
    * **Pré-condições:** Um usuário válido já cadastrado.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  Preencher "Email Address" com um e-mail **válido e registrado**.
        3.  Preencher "Password" com uma **senha incorreta**.
        4.  Clicar em "Login".
    * **Resultados Esperados:**
        * Mensagem de erro "Your email or password is incorrect!" é exibida.
        * O usuário permanece na página de Login.

4.  **Nome do Caso de Teste:** Login com Campos Vazios
    * **Objetivo:** Verificar a validação de campos vazios no formulário de login.
    * **Pré-condições:** Acesso à página de Login.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  Deixar "Email Address" vazio e preencher "Password".
        3.  OU preencher "Email Address" e deixar "Password" vazio.
        4.  OU deixar ambos os campos vazios.
        5.  Clicar em "Login".
    * **Resultados Esperados:**
        * Mensagens de erro de validação do navegador/HTML5 são exibidas (ex: "Please fill out this field").
        * O usuário permanece na página de Login.

5.  **Nome do Caso de Teste:** Login com Formato de Email Inválido
    * **Objetivo:** Verificar se o sistema valida o formato do endereço de e-mail no login.
    * **Pré-condições:** Acesso à página de Login.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  Preencher "Email Address" com um formato inválido (ex: "teste@.com", "teste", "teste.com").
        3.  Preencher "Password" com qualquer senha.
        4.  Clicar em "Login".
    * **Resultados Esperados:**
        * Mensagem de erro de validação de e-mail é exibida (geralmente do navegador).
        * O usuário permanece na página.

6.  **Nome do Caso de Teste:** Teste de Capitalização (Case Sensitivity) da Senha
    * **Objetivo:** Confirmar se a senha é case-sensitive.
    * **Pré-condições:** Um usuário válido com uma senha que contenha letras maiúsculas e minúsculas.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  Preencher "Email Address" com e-mail válido.
        3.  Preencher "Password" com a senha correta, mas com a **capitalização errada** (ex: "Senha123" vs "senha123").
        4.  Clicar em "Login".
    * **Resultados Esperados:**
        * Mensagem de erro "Your email or password is incorrect!" é exibida.
        * O usuário permanece na página de Login.

7.  **Nome do Caso de Teste:** Fluxo de "Forgot Your Password?"
    * **Objetivo:** Verificar a funcionalidade de recuperação de senha (apenas a navegação inicial, sem o fluxo completo de e-mail).
    * **Pré-condições:** Acesso à página de Login.
    * **Passos:**
        1.  Acessar `https://www.automationexercise.com/login`.
        2.  Clicar no link "Forgot your password?".
    * **Resultados Esperados:**
        * Usuário é redirecionado para a página de recuperação de senha.
        * Um campo para inserir o e-mail (ou similar) é visível.

### Organização dos Cenários em Arquivos para Robot Framework (Manutenção Futura)

Mesmo para teste manual, pensar na estrutura do Robot Framework ajuda na organização e facilita uma eventual migração para automação.

* **`tests/functional/login_tests.robot`**: Conteria os cenários de login (`Cenário: Login com Credenciais Válidas`, `Cenário: Login com Email Não Registrado`, etc.). Cada Test Case seria um dos cenários detalhados acima.
* **`tests/functional/registration_tests.robot`**: Conteria os cenários de cadastro (`Cenário: Cadastro Completo de Novo Usuário com Sucesso`, `Cenário: Tentativa de Cadastro com Email Já Registrado`, etc.).
* **`resources/test_data/users.robot`**: Este arquivo de recurso pode conter variáveis com dados de teste para usuários (ex: `${VALID_USER_EMAIL}`, `${VALID_USER_PASSWORD}`, `${INVALID_USER_EMAIL}`). Para testes manuais, isso seria mais como uma referência rápida para o testador.
* **Documentação Adicional**: Um documento à parte (ou no próprio `README.md` do projeto) listando os dados de usuários pré-cadastrados, URLs e outras informações de ambiente que seriam usadas nos testes manuais.

Essa estrutura, mesmo não sendo diretamente executada como código, fornece um modelo mental claro para a execução manual, além de preparar o terreno para uma futura automação, caso seja necessário.
